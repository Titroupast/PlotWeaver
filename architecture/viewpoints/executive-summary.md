# PlotWeaver Phase 1 部署执行摘要

## 业务目标
交付一个稳定、可追踪、可人工介入的小说续写平台：用户可在 Web 端发起与审阅 run，后端可稳定生成并保存结构化结果。

## 关键决策
1. 前端使用 Vercel，保证预览与上线效率。
2. 后端使用独立容器服务，保证运行控制与可扩展性。
3. 数据层采用 Supabase PostgreSQL，承载租户隔离与核心状态。
4. 上线前必须通过 Contract/API/E2E 门禁。

## 本阶段不做
1. 多区域双活与复杂灾备拓扑。
2. 重平台化的基础设施抽象层。
3. 超出现有 run 状态机的复杂分布式编排系统。

## 成功指标
1. 常规变更部署 Lead Time 小于 20 分钟。
2. P1 故障中位恢复时间小于 30 分钟。
3. run 创建与审阅关键路径可用性高于 99.5%。
