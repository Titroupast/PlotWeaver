# 04 数据库迁移与恢复

## 迁移策略
1. 所有 Schema 变更必须通过 Alembic 版本脚本。
2. 先在 staging 环境执行迁移验证。
3. 生产迁移需在受控发布窗口执行。

## 迁移命令
```bash
alembic upgrade head
```

## 备份策略
1. 每日自动备份。
2. 高风险迁移前额外做发布前快照。
3. 保留期建议：staging 7 天，prod 30 天以上。

## 恢复策略
1. 优先前向修复（forward-fix）迁移。
2. 仅在严重损坏场景使用快照恢复。
3. 恢复后补跑必要的应用迁移与配置。

## 验证步骤
1. 检查健康接口。
2. 验证 project/chapter/run 列表 API。
3. 验证 run 事件流查询可用性。
