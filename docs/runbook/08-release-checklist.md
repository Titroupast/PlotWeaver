# 08 发布检查清单

## 发布前
1. CI PR 门禁全部通过。
2. staging 部署完成。
3. 契约/API/E2E 门禁全部绿色。
4. 待执行迁移已完成评审。
5. 生产备份快照已确认可用。

## 发布中
1. 部署 API 镜像。
2. 执行 `alembic upgrade head`。
3. 在 Vercel 部署或提升 Web 版本。
4. 执行冒烟检查：
   - `/api/v1/health/live`
   - `/app/projects`
   - run 详情页与 review 页

## 发布后
1. 观察错误看板 30 分钟。
2. 确认无 P1/P2 告警。
3. 记录发布说明与回滚参考。

## 回滚判定规则
满足任一条件立即回滚：
1. API 不可用超过 5 分钟。
2. 核心 run 接口持续失败。
3. 怀疑出现数据一致性问题。
