# 06 告警与值班

## 严重级别
| 级别 | 描述 | 响应要求 |
| --- | --- | --- |
| P1 | 生产不可用或核心数据链路严重故障 | 值班人员立即响应 |
| P2 | 服务降级但未完全中断 | 工作时间内响应 |
| P3 | 轻微问题或提示类告警 | 纳入待办跟进 |

## 告警触发条件
### P1
1. API 健康检查连续失败超过 5 分钟。
2. 数据库连接错误超过阈值。
3. Run 创建接口在 5 分钟窗口失败率超过 20%。

### P2
1. SSE 降级比例在 15 分钟内超过 20%。
2. 连续两次部署失败。
3. 4xx/5xx 异常趋势持续 30 分钟。

## 通知渠道
1. 团队聊天工具值班频道。
2. P1 级别使用 Pager 升级通知。
3. P2/P3 使用邮件摘要同步。

## 事件处理步骤
1. 确认并接管告警。
2. 评估影响范围。
3. 判定回滚或前向修复。
4. P1 事件每 15 分钟更新一次状态。
5. 24 小时内产出事故复盘总结。
